Saya akan membahas tentang PHP, dimana pasti Anda yang akan teringat antara Laravel atau WordPress, tanpa bermaksud menyudutkan framework PHP lainnya. Mengingat status PHP sebagai bahasa pemrograman Backend, tentu kurang lengkap jika tidak membahas seputar API. Setiap backend membutuhkan API untuk menghubungkan "cinta terlarang" Backend dengan Frontend. Terlarang bagi penganut sekte monolith garis keras yang berpendapat bahwa satu bahasa harus digunakan untuk Backend dan Frontend.

Apa itu API Resources?
----------------------

Setelah Anda memahami secara umum tentang API, sekarang mari kita bahas topik yang lebih spesifik. Saya akan menjelaskan tentang API Resource yang secara sederhana dapat dianalogikan seperti "cetakan es batu di freezer". Hey ! Jangan tergoda dulu, karena saya ingin menjelaskan lebih detail. Bayangkan jika Anda membuat es hanya menggunakan plastik transparan atau kantong plastik hitam yang diletakkan di freezer - bentuknya pasti akan kacau dan tidak beraturan. Namun, jika Anda menggunakan cetakan, hasilnya akan terlihat rapi, konsisten, dan tidak berantakan.

Jadi, dapat disimpulkan fungsi dari API Resource

1.  Merapikan data Saya memberikan contoh kasus yang umum digunakan seperti memilih data mana yang ingin Anda tampilkan. Selain itu, data sensitif seperti password dapat Anda sembunyikan. Kemudian, format tanggal dan waktu juga bisa Anda sesuaikan sesuai kebutuhan.
2.  Konsistensi API Resources dapat menciptakan konsistensi pada API yang akan Anda berikan dari Laravel ke frontend. Hal ini tercapai karena semua data yang keluar akan memiliki struktur yang sama sehingga memudahkan pengembangan di sisi frontend. Dengan demikian, penggunaan API Resources tidak hanya memudahkan programmer lain yang menggunakan API Anda, tetapi juga mengurangi kemungkinan terjadinya kesalahan format data.

Saya sarankan agar Anda menggunakan API Resource jika ingin kode Laravel Anda tetap terstruktur dengan baik. Dengan begitu, Anda tidak perlu mengkhawatirkan masalah format data. Jika anda masih belum paham kegunaannya API Resource , tidak masalah ulang lagi baca diatas sampai paham betul atau silahkan meluncur ke dokumentasi Laravel resmi dibawah ini.

<https://laravel.com/docs/11.x/eloquent-resources>

Cara Membuat API Resource di Laravel 11
---------------------------------------

### Langkah 1: Membuat Resource

Pertama, buka terminal atau command prompt jalankan perintah

```
php artisan make:resource UserResource

```

### Langkah 2: Mengatur Format Data

Buka file `UserResource.php` lalu silahkan racik format API terbaik menurut anda atau tim pada fungsi `toArray` .

```
<?php

namespace App\\Http\\Resources;

use Illuminate\\Http\\Request;
use Illuminate\\Http\\Resources\\Json\\JsonResource;

class UserResource extends JsonResource
{
    public function toArray(Request $request): array
    {
        return [
            'id' => $this->id,
            'nama' => $this->name,
            'email' => $this->email,
            // Format tanggal agar mudah dibaca
            'tanggal_daftar' => $this->created_at->format('d M Y'),
            // Hitung umur user dari tanggal lahir
            'umur' => $this->tanggal_lahir->age
        ];
    }
}

```

### Langkah 3: Penggunaan Resource di Controller

Tahap terakhir yaitu pengunaannya tentu pasang ~~C4~~ API Resource di Controller yang sudah dimiliki

```
<?php

namespace App\\Http\\Controllers;

use App\\Models\\User;
use App\\Http\\Resources\\UserResource;

class UserController extends Controller
{
    public function show($id)
    {
        // Cari user berdasarkan ID
        $user = User::find($id);

        // Kirim data menggunakan Resource
        return new UserResource($user);
    }

    public function index()
    {
        // Ambil semua user dan gunakan pagination
        $users = User::paginate(10);

        // Kirim collection data menggunakan Resource
        return UserResource::collection($users);
    }
}

```
